# Contributing

Thank you for your interest in contributing to this project.

## Guidelines

- Keep the implementation simple, readable, and production-oriented.
- Maintain consistent naming and file structure.
- Update documentation when changing user-facing behavior.
- Prefer modular improvements over large unstructured changes.

## Commit style

Recommended commit prefixes:
- `feat:` new functionality
- `fix:` bug fixes
- `docs:` documentation changes
- `refactor:` internal improvements
- `chore:` maintenance tasks
